# AI Growth OS Beta Release

## Current Capability

AI Growth OS Beta now supports the complete local-first experience:

Knowledge → Practice → Case Challenge → Evidence → Capability Growth → Project → Profile.

Users can learn foundations, submit rule-evaluated Practice work, solve existing
Cases, see evidence projected into the Capability Map and Profile, record three
Project milestones, and view an isolated public Demo Profile. Practice, Project,
and feedback records persist locally in independent versioned sidecars.

## Known Limitations

- There is no account, backend, cloud synchronization, or cross-device history.
- Practice evaluation is deterministic and rule based, not semantic AI review.
- Project Evidence records milestone completion only; it is not project management.
- Feedback remains on the current browser until the tester shares it manually.
- Demo Profile is static showcase data and is not a public publishing system.
- Mastery remains driven by the existing completed Attempt algorithm; Practice
  evidence is a read-only MVP projection.

## User Testing Goals

Test with 5–20 users and observe, without coaching:

1. Can they explain what the product is and who it is for after the first screen?
2. Do they start and complete their first Practice?
3. Do they continue from Practice into a Case Challenge?
4. Does the Capability Map create motivation or confusion?
5. Which Skills and content attract attention, and which are ignored?
6. Can they understand and record Project progress?

Capture content problems, Case difficulty, and feature suggestions through the
local Feedback page, then export findings manually during research sessions.

## Future Roadmap

Development is frozen pending real-user evidence. The following are explicitly
deferred: AI Mentor, enterprise features and certification, cloud backend,
accounts and synchronization, full Skill Graph migration, and expansion to 70
Skills. Resume development only when observed user behavior establishes the next
highest-value problem.
